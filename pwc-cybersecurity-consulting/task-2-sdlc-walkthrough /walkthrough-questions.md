# SDLC Walkthrough Questions

## Governance and Planning

1. Who is responsible for security oversight during the SDLC?
2. Are security requirements documented during project initiation?
3. How are security roles assigned to development projects?

## Risk Assessment

4. Is a formal risk assessment performed before development begins?
5. How are identified risks tracked and mitigated?

## Development and Testing

6. Are security requirements incorporated into system design?
7. Is security testing performed before deployment?
8. Are test results documented and reviewed?

## Change Management

9. Are production changes formally approved?
10. Is there evidence of change testing before implementation?

## Monitoring and Maintenance

11. How are security issues monitored after deployment?
12. How are security patches evaluated and implemented?

## System Retirement

13. How is sensitive information removed from retired systems?
14. Is there a documented disposal process for hardware and software?
